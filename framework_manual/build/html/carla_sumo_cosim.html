CARLA and SUMO Setup

This section explains how to adapt the CARLA–SUMO co-simulation framework so it can interface with a network simulator through ZeroMQ. CARLA has supported native SUMO co-simulation since version 0.9.8, enabling synchronized actions and state between both simulators. Here, we extend this functionality to broadcast CARLA sensor data to an external network simulator.

All required integration files are located in:

Path/To/CARLA/Co-Simulation/Sumo

Make a copy of that directory and rename it to something like:

sumo_omnetpp

This new directory will hold all modifications needed to connect CARLA, SUMO, and Artery.

carla_simulation.py

ZeroMQ (ZMQ) is the central component that enables communication between CARLA and Artery. Begin by modifying:

Path/To/CARLA/Co-Simulation/sumo_omnetpp/sumo_integration/carla_simulation.py

ZMQ Server and Router Thread

Next, define the ZMQ server and internal router. The server publishes sensor data externally, while the router listens to in-process channels and forwards messages.

start_zmq_server and router methods
 1 def start_zmq_server(self):
 2     self.context = zmq.Context()
 3     self.socket = self.context.socket(zmq.PUB)
 4     self.socket.bind(f"tcp://{self.host}:{self.zmq_port}")
 5
 6     # Thread that forwards messages to the OMNeT++ side
 7     self.thread = threading.Thread(target=self.router)
 8     self.thread.start()
 9
10 def router(self):
11     router_socket = self.context.socket(zmq.SUB)
12     router_socket.bind("inproc://internalstuff")
13     router_socket.subscribe("")
14
15     poller = zmq.Poller()
16     poller.register(router_socket, zmq.POLLIN)
17
18     try:
19         while not self.stop_event.is_set():
20             try:
21                 socks = dict(poller.poll(100))  # timeout 100 ms
22             except zmq.error.ContextTerminated:
23                 break
24
25             if router_socket in socks and socks[router_socket] == zmq.POLLIN:
26                 frames = router_socket.recv_multipart()
27                 self.socket.send_multipart([frames[0], frames[1]])
28     finally:
29         router_socket.close()

Sensor Integration

Before adding sensors, note that camera-based sensors are not suitable for this architecture. Their frames must be processed externally, and transmitting raw images over ZMQ is inefficient. Instead, use lightweight numeric sensors such as:

  • sensor.other.obstacle

  • sensor.lidar.ray_cast

  • sensor.other.radar

Sensor Callback

This function receives sensor data, formats the message, and publishes it through ZMQ.

sensor_callback method
 1 def sensor_callback(self, sensor_msrmnt, vehicle_id, sensor_id, zmq_socket):
 2     obst_msg = dict()
 3     obst_msg["message_type"] = "OBSTACLE_DATA"
 4     obst_msg["vehicle_id"] = int(list(self.sumo2carla_ids.keys())[list(self.sumo2carla_ids.values()).index(vehicle_id)])
 5     obst_msg["sensor_id"] = sensor_id
 6     obst_msg["timestamp"] = sensor_msrmnt.timestamp-self.start_time
 7     obst_msg["distance"] = sensor_msrmnt.distance
 8
 9     print("    sensor", sensor_id, "@ car ", obst_msg["vehicle_id"], "distance =", sensor_msrmnt.distance, "timestamp =", sensor_msrmnt.timestamp-self.start_time)
10
11     # Publish data via ZMQ
12     if vehicle_id in self.sumo2carla_ids.values():
13         carla2sumo_id = str(
14             list(self.sumo2carla_ids.keys())[list(self.sumo2carla_ids.values()).index(vehicle_id)]
15         )
16     else:
17         return
18
19     topic = carla2sumo_id + ".obst" + str(sensor_id)
20     zmq_socket.send_multipart([topic.encode("utf-8"), json.dumps(obst_msg).encode("utf-8")])

Actor Destruction

destroy_actor method
 1 def destroy_actor(self, actor_id):
 2     """
 3     Destroys the given actor.
 4     """
 5     actor = self.world.get_actor(actor_id)
 6     # Destroy all sensors attached to this actor
 7     cmd_batch = []
 8     for idx, sensor in enumerate(self.sensor_list):
 9         if sensor.parent.id==actor_id:
10             self.sensor_list[idx].stop()
11             self.sensor_router_socket_list[idx].close()
12             cmd_batch.append(carla.command.DestroyActor(sensor.id))
13     if cmd_batch:
14         self.client.apply_batch_sync(cmd_batch, False)
15
16     if actor is not None:
17         return actor.destroy()
18     return False

Summary

In this section, you:

  • Extended the default CARLA–SUMO co-simulation

  • Added ZeroMQ support for external communication

  • Attached obstacle sensors to CARLA vehicles

  • Implemented a callback to publish sensor data

  • Updated the synchronization script to propagate vehicle IDs

  • Added support for configurable ZMQ ports

With this setup, CARLA and SUMO can now send obstacle measurements to a network simulator such as Artery.
